1. 比如添加一条到主机(12.23.34.45)的路由(通过路由器192.168.1.1)：
route add -host 12.23.34.45 gw 192.168.1.1

路由表中就添加了一个条目：
Destination     Gateway         Genmask         Flags Metric Ref    Use Iface
12.34.56.78     192.168.1.1     255.255.255.255 UGH   0      0        0 wlan0

就很明了了：
首先根据flags中的H，说明这是一条到主机的路由（而不是到某个网络的路由），中的G，说明
还需要通过一个网关就是路由器192.168.1.1来转发数据包（间接交付）,而且因为是到主机的路由，所以
Genmask就固定为255.255.255.255了（255.255.255.255 for a host destination and 0.0.0.0 for the default route.）。

2. 再比如添加一条到某个网络（10.20.30.48，子网掩码255.255.255.248）的路由（通过路由器192.168.1.1转发）：
route add -net 10.20.30.48 netmask 255.255.255.248 gw 192.168.1.1

路由表中就添加了一个条目：
Destination     Gateway         Genmask         Flags Metric Ref    Use Iface
10.20.30.48     192.168.1.1     255.255.255.248 UG    0      0        0 wlan0

意思也很明了：
首先根据flags，发现没有H，说明是到某个网络的路由，发现G说明还是间接交付（要通过路由器转发），
到哪个网络的路由呢？
我们用(10.20.30.48 & 255.255.255.248)就得到了这个目标网络了假设是n1.
当我们收到一个IP地址a,我们首先用(a & Genmask)，得到一个网络号n2，然后和n1比较，如果n1 == n2，那么这条
路由表项就发挥作用了，它指明凡是发往该网络的数据包都通过192.168.1.1转发。如果n1 != n2，说明不是发给我们
这个网络的，于是查找下一个条目。

3. 最后比如添加一个默认路由，全部发给192.168.1.1路由器转发：
route add -net default gw 192.168.1.1 

路由表中就添加了一个条目：
Destination     Gateway         Genmask         Flags Metric Ref    Use Iface
0.0.0.0         192.168.1.1     0.0.0.0         UG    202    0        0 wlan0
意思也明白了：
Destination为0.0.0.0就表示默认路由了，那个G就表示也需要间接交付，通过路由器（192.168.1.1转发）

4. 路由表中还会出现这样的条目：
Destination     Gateway         Genmask         Flags Metric Ref    Use Iface
192.168.1.0     0.0.0.0         255.255.255.0   U     202    0        0 wlan0

其实这个也不难理解：
首先看flags,没有G,说明是直接交付！没有H说明是到网络的路由！到哪个网络呢？由Destination和Genmask字段
就不难计算出来是192.168.1这个网段，Getway：0.0.0.0也就是发给该网络中的所有主机，因为你一定也在192.168.1这个
网段上，所以就算是群发了，目的主机一定是可达的。所以，凡是发送到192.168.1.xxx的数据包，本路由都可以
直接交付了！
